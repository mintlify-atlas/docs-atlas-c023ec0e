## What is CollabMD?

CollabMD is a self-hosted CLI tool that serves any directory of Markdown files as a realtime collaborative web application. It is built for teams that want the power of a collaborative editor without giving up their local filesystem as the source of truth.

Point it at your existing vault:

```bash
npx collabmd@latest ~/my-vault --no-tunnel
```

Everyone with access to the URL can open the same files, edit simultaneously, and see each other's cursors — just like Google Docs, but backed by plain `.md` files on your disk.

## How it works

CollabMD runs a lightweight Node.js server that:

1. Reads your Markdown files from disk at startup
2. Opens a WebSocket-based collaboration room per file using Yjs
3. Serves a browser-based editor (CodeMirror + React) and live preview
4. Writes changes back to disk as plain text as you type

Your filesystem is always the source of truth. CollabMD does not move, rename, or delete files unless you explicitly do so inside the app.

## Best fit for

- Collaborating on an existing Obsidian-style vault without migrating files
- Reviewing RFCs, product docs, architecture notes, and runbooks in real time
- Commenting on drafts and diagrams with anchored threads instead of side-channel feedback
- Sharing Markdown-heavy knowledge bases with remote teammates
- Giving browser access to collaborators who do not use your local Markdown setup

## Current limitations

<Warning>
  CollabMD is a single-instance deployment only. Room state is kept in-process and is not shared across replicas.
</Warning>

- **Single instance**: collaboration state lives in memory; horizontal scaling is not supported
- **OIDC auth**: reserved for a future release — `--auth oidc` is not yet available
- **Excalidraw comments**: source-anchored comments do not apply to `.excalidraw` files
- **Windows**: supported via WSL2, not native Windows execution
