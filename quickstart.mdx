## Prerequisites

- **Node.js 24+** — required for `npx` usage. Check your version with `node --version`.
- A folder of Markdown files on your machine (your "vault").

<Steps>
  <Step title="Run CollabMD with npx">
    No installation needed. Run CollabMD directly against your vault:

    ```bash
    npx collabmd@latest ~/my-vault --no-tunnel
    ```

    Replace `~/my-vault` with the path to any folder containing Markdown files. The current directory is used if you omit the path.

    <Note>
      The `--no-tunnel` flag disables the Cloudflare Tunnel. Tunnel support is on by default when `cloudflared` is installed — it exposes your session over a public URL. Use `--no-tunnel` for local-only access until you are ready to share.
    </Note>
  </Step>

  <Step title="Confirm startup output">
    Once the server is ready, you will see output similar to:

    ```text
    ╔══════════════════════════════════════╗
    ║           CollabMD v0.1.19           ║
    ╚══════════════════════════════════════╝

      Vault:  /home/you/my-vault (12 markdown files)
      Local:  http://localhost:1234
      PlantUML: https://www.plantuml.com/plantuml
      Auth:   none
      Tunnel: disabled

      Ready for collaboration. Press Ctrl+C to stop.
    ```
  </Step>

  <Step title="Open the workspace in your browser">
    Navigate to:

    ```text
    http://localhost:1234
    ```

    You will see a file explorer sidebar on the left, the Markdown editor in the center, and a live preview panel on the right.
  </Step>

  <Step title="Edit a file">
    Select any `.md` file from the sidebar. Start typing — the live preview updates as you type. You can create new files, rename them, and organize folders directly in the browser.
  </Step>

  <Step title="Share with a collaborator (optional)">
    To invite someone else to edit in real time, start CollabMD with password auth so the session is protected before exposing it:

    ```bash
    npx collabmd@latest ~/my-vault --auth password
    ```

    CollabMD will print a generated password and a tunnel URL (if `cloudflared` is installed). Share both with your collaborator. See the [CLI reference](/installation) for `--auth-password` if you want to set your own password.
  </Step>
</Steps>

## Next steps

- **Permanent install** — prefer a globally available `collabmd` command? See [Installation](/installation) for Homebrew, source, and Docker options.
- **Auth and sharing** — run `collabmd --auth password` and share the printed URL and password.
- **Custom port** — use `--port 3000` if port 1234 is taken.
